Project Documentation

A comprehensive guide to the evolution of KeyR, detailing major architectural shifts, performance milestones, and the roadmap to automation excellence.

v0.1.0 Beta (Initial Release)

The Foundation

Version 1.0 established the core KeyR philosophy: a lightweight, scripting-free automation tool that anyone can use. The goal was to eliminate the steep learning curve of traditional macro tools by providing a "Record and Play" experience that just works. It introduced the proprietary .tt2 encrypted macro format to protect user privacy and workflow integrity.

Unified Macro Engine The heart of KeyR. It captures raw mouse coordinates and virtual keyboard scan codes into a single, high-fidelity timeline. Unlike other recorders that only capture clicks, KeyR records the spatial movement and timing of every interaction, ensuring 1:1 playback precision.
Secure (.tt2) Encryption All macros are saved in a proprietary binary format protected by AES-128 encryption. This ensures that your automated workflows—which might contain sensitive login steps or private data—cannot be read or modified by text editors or unauthorized users.
Variable Replay Speed (0.1x – 10x) A dynamic timing engine that allows users to warp time. By decoupling the captured timing from the playback clock, you can slow down complex tasks to 10% speed for debugging, or speed up repetitive data entry by up to 1000% without losing input accuracy.
Infinite & Counted Looping Built-in support for automation loops. You can specify a exact number of times for the macro to run, or enable "Continuous Mode" for tasks that need to run indefinitely until manually stopped.
Global System Hotkeys System-wide hooks for recording and playback control. By using F8 (Record/Stop) and F10 (Play/Panic Stop), you can control KeyR even when it is minimized in the system tray or when you are focused on another application.
UI Persistence & Layout Settings The application remembers your workspace. It automatically saves the "Always-on-Top" state and your exact window coordinates, so the controller is always exactly where you left it upon the next launch.
Zero-Install Portability A standalone executable (~600KB) that requires no installation, registry changes, or external dependencies (other than .NET). It is designed to be carried on a USB drive and used across any Windows machine.
v0.1.1 Beta (Stability & UX Refining)

Stability & UX Refining

The 1.1 update focused on the "Quality of Life" improvements requested by early adopters. It addressed critical edge cases in loop logic and introduced the first phase of workspace persistence to ensure a professional, reliable environment for long-running automations.

Interruptible Loop Architecture Engineered a more responsive cancellation token system that allows users to halt active playback immediately. By decoupling the input stream from the main lifecycle, the "Panic Stop" (F10) now works instantly even during high-frequency mouse clicks.
Workspace Intelligence Cache Implemented automatic coordinate and preference caching for the main window. The application now remembers its exact last known position on the user's desktop, restoring it upon relaunch to maintain a consistent multi-monitor workspace.
Adaptive Always-on-Top View Added a dedicated toggle to keep the macro controller visible over other applications. This is essential for monitoring macro progress while playing full-screen games or using professional software that usually hides background windows.
Persistent Loop Configuration The desired loop count is now saved into the local settings JSON. This prevents users from having to re-enter their complex repetition patterns every time the application is closed or updated.
v0.1.2 Beta (Performance Milestone)

Low-Level Optimization

Version 1.2 represents a fundamental shift from feature expansion to low-level hardware optimization. The primary objective was to minimize the application's runtime footprint while maximizing timing precision for high-speed macro replay.

Allocation-Free Playback Loop By caching native Win32 input structures and avoiding repeated memory allocations during the playback loop, v0.1.2 virtually eliminates Garbage Collection pauses during critical macro execution. This results in stutter-free playback.
Micro-Precision Clocking Transitioned to a tighter spin-wait mechanism (10-cycle accuracy) for micro-second level delay precision. This ensures that even the fastest rapid-fire macros are replayed with exactly the same timing as the original recording.
Binary Trimming & Stripping Leveraged advanced IL stripping and dead-code elimination to reduce the executable size to its most essential components (~600KB). This ensures nearly instantaneous application startup and minimal RAM usage.
Optimized Thread Scheduling Macros now run on high-priority threads to prevent Windows from deprioritizing the playback during CPU-intensive tasks, ensuring your automation remains consistent even when the PC is under heavy load.
v0.2.0 Beta (The Smart Logic Update)

Intelligent Automation

Version 2.0 transforms KeyR from a simple "playback robot" into an intelligent "decision-making assistant." By introducing a background environmental scan engine, KeyR can now interact with the visible state of your computer in real-time. This allows for complex workflows that wait for specific events before proceeding.

Smart Auto-Restart Engine A parallel background thread that monitors screen conditions while the macro is running. If your specified conditions are met, the engine can instantly "Rewind" the macro back to loop 1, creating a truly reactive automation loop that responds to external triggers.
On-Screen Condition Detection
Set up multiple triggers based on what is happening on your monitor in real-time:
Image Detection: Scans a specific region (or full screen) for a reference image with adjustable color tolerance and pixel-perfect comparison.
Text Detection (OCR): Uses advanced native Windows Character Recognition to "read" text inside hidden and active windows to trigger macro restarts.
Time Intervals: Intelligent timers designed to ensure periodic restarts or status checks regardless of what is happening on screen.
Match Logic: ANY vs ALL Sophisticated boolean evaluation for complex tasks. Choose "Match ANY" to restart if any one of your conditions becomes true, or "Match ALL" to ensure a multi-step prerequisite is met before the macro triggers.
Logic Toggles: Repetitive vs Sequential
Repetitive (Pulse): The macro restarts immediately and repeatedly as long as the condition is visible. Perfect for rapid-fire "spam" loops.
Sequential (Smart): The macro restarts once, then waits for the condition to disappear before allowing another restart. This prevents accidental multi-restarts on static images.
Adjustable Polling Interval Full control over performance. You can set the engine to scan the screen every 100ms for high-speed reactions, or set it to a longer interval to save CPU resources during long-term background monitoring tasks.
Single Instance Hardware Lock To prevent input conflicts and system instability, KeyR v0.2.0 includes a system-level mutex lock that ensures only one instance of the application can run at a time.
v0.3.0 Beta (The UI Overhaul Update)

Settings & Theme Engine Overhaul

Version 3.0 represents a massive step forward for the application's configuration user experience. The update introduces a brand new, dedicated Settings Window, complete with robust flat-color themes and highly responsive interface components designed to make tuning your automation as intuitive as possible.

Modern Flat Themes The underlying Theme Engine was fundamentally rebuilt. Legacy UI gradients were entirely stripped out and replaced with deep, solid color palettes, producing a significantly sharper, more premium Dark Mode aesthetic.
Centralized Settings Window A brand new window instance dedicated solely to configuring KeyR. It cleanly isolates preferences from the main launcher interface and provides ample room for advanced controls, import/export capabilities, and accessibility toggles.
Real-Time Search Bar Users can now instantly filter down the extensive settings list. The robust new search bar instantly narrows down the UI layout to the specific toggles or inputs matching the query, making configuration nearly instantaneous.
Redesigned Setting Tiles Configuration cards and inputs were graphically reimagined. For example, the Quick-Move utility was completely visually overhauled into an elegant, interactive 3x3 directional tile layout, making screen-snapping much more user-friendly.
Quality-of-Life Tooltips Reinvented the helper tooltips to be highly accessible and dynamic. Tooltips now react in real-time to button clicks (e.g., dynamically swapping from "Always On Top is Off" to "Always On Top is On" the moment it is toggled) without requiring a mouse re-hover.
Bug Fixes & Polish Crushed several persistent visual and layout bugs, most notably a critical fix for the search bar cursor that was rendering entirely invisible when users activated the pitch-black Dark Mode theme.
v0.3.1 Beta (QoL & Refinements)

Settings Polishing

Version 3.1 introduces targeted quality-of-life improvements to streamline user interaction with Settings and Smart Automations.

Foreign Macro Conversion New advanced system allowing other macro files (InformaalTask and TinyTask) to be parsed, imported, and directly converted into KeyR's encrypted JSON framework (.tt2) effortlessly. Find the tools in Settings under "Advanced".

Theme Dropdown Evolution The dark theme checkbox was upgraded into a distinct, dropdown component to unify styling with other dropdowns, like Condition Types.
Contextual UI Logic The 'Restricted' toggle is now context-aware, making it visible only when Sequential Restart Logic is active, completely decluttering the settings menu.
v0.3.2 Beta (Engine Overhaul)

Native Input Engine & Recording Timer

Version 3.2 represents the most significant backend overhaul in KeyR's history. The entire recording subsystem was rebuilt from the ground up using native Win32 low-level hooks, replacing the third-party MouseKeyHook library for input capture. This eliminates an entire class of missed-input bugs and enables capture of previously invisible actions like XButton1/XButton2 mouse buttons, horizontal scroll wheels, and extended keyboard keys (arrows, navigation cluster, numpad enter).

Native Win32 Recording Hooks Recording now uses WH_MOUSE_LL and WH_KEYBOARD_LL hooks installed directly via SetWindowsHookEx, bypassing the managed MouseKeyHook wrapper. This captures every raw input message at the OS level before any application processes it, ensuring zero missed events even during rapid-fire input sequences.
Complete Input Coverage
The new hook callbacks process the full spectrum of Windows input messages:
Mouse: WM_MOUSEMOVE, WM_LBUTTONDOWN/UP, WM_RBUTTONDOWN/UP, WM_MBUTTONDOWN/UP, WM_XBUTTONDOWN/UP, WM_MOUSEWHEEL, WM_MOUSEHWHEEL
Keyboard: WM_KEYDOWN, WM_KEYUP, WM_SYSKEYDOWN, WM_SYSKEYUP — with LLKHF_EXTENDED flag preservation for arrow keys, navigation keys, and numpad enter.
Live Recording Timer A new DispatcherTimer displays elapsed recording time in the title bar in real-time (MM:SS format), mirroring the countdown timer shown during playback. This gives users immediate visual feedback on how long their recording has been running without needing to check an external clock.
3-Tier Precision Playback Timing
The playback delay engine was redesigned with a tiered strategy for optimal CPU usage and timing accuracy:
Tier 1 (>15ms): Thread.Sleep(1) — yields the CPU completely, ideal for long delays between actions.
Tier 2 (2–15ms): Thread.Sleep(0) — yields to same-priority threads only, keeping the thread warm.
Tier 3 (<2ms): Thread.SpinWait(10) — active spin for sub-millisecond precision, ensuring zero-drift playback for rapid macro sequences.
Extended Key Flag Preservation The raw KBDLLHOOKSTRUCT.flags field is now captured and stored in each MacroEvent. During playback, this flag is combined with a known-extended-key lookup table to ensure correct KEYEVENTF_EXTENDEDKEY behavior, fixing issues where arrow keys, Delete, Home/End, or numpad Enter would replay as incorrect keycodes.
Backward Compatibility The new MacroEvent schema is a strict superset of the previous format. Old .tt2 macro files from any prior version load and play back correctly without conversion — new fields like IsExtendedKey and ScrollDelta are safely zero-initialized when absent from older recordings.
v0.3.3 Beta (.NET Bootstrapper)

Dependency Checks

Version 3.3 introduces an intelligent startup bootstrapper designed to dramatically improve the onboarding experience for new users.

Smart First-Boot Diagnostics Since KeyR relies on the .NET 9 Desktop Runtime, older versions would fail silently or show generic Windows error dialogs if the runtime wasn't installed. The app now natively hooks into the startup sequence to probe for required assemblies (like System.Drawing.Common) before trying to initialize the UI.
Themed Information Dialogs Designed a completely new custom `ThemedInfoBox` component to handle OS messages, warnings, and dependency checks elegantly, abandoning rigid Windows MessageBox forms in favor of fully custom styling.
